BACKEND LARAVEL CREE

Dossier :
portfolio-backend

URL locale :
http://127.0.0.1:8000

Connexion admin :
http://127.0.0.1:8000/login

Compte admin de depart :
email : admin@example.com
mot de passe : password

Pages admin :
- /admin : dashboard KPI
- /admin/projects : gestion des projets
- /admin/messages : messages du formulaire contact

Routes API :
- GET /api/projects
- GET /api/projects/{slug}
- POST /api/contact
- POST /api/visits

Commande pour lancer le backend :
cd portfolio-backend
php artisan serve

Commande pour tester le backend :
cd portfolio-backend
php artisan test

Commande utile pour afficher les images uploadees :
cd portfolio-backend
php artisan storage:link

Important pour les emails :
Pour le moment, MAIL_MAILER=log.
Cela veut dire que Laravel n'envoie pas encore vers une vraie boite email.
Il ecrit l'email dans les logs pour tester sans risque.

Quand tu voudras recevoir les vrais emails, on modifiera dans portfolio-backend/.env :
- MAIL_MAILER
- MAIL_HOST
- MAIL_PORT
- MAIL_USERNAME
- MAIL_PASSWORD
- MAIL_FROM_ADDRESS
- ADMIN_EMAIL

Prochaine etape conseillee :
Connecter le composant Projects.tsx du front-end Next.js a GET /api/projects.
